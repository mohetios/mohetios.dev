# Universal workflow

## A. Understand

1. خواسته‌ی کاربر را به operation تبدیل کن.
2. surface و مخاطب را تشخیص بده.
3. سطح مداخله را مشخص کن.
4. منابع factual را جدا کن.
5. context محلی مرتبط را پیدا کن؛ context نامرتبط را نخوان.

## B. Freeze

یک inventory کوتاه بساز:

```text
Facts
Status and dates
Ownership and attribution
Numbers
URLs and identifiers
Code and placeholders
Quotes
Uncertainty and limitations
```

## C. Shape

- پیام مرکزی متن را در یک جمله مشخص کن.
- ترتیب reasoning را بساز.
- repetition، مقدمه‌ی generic و sectionهای بی‌کارکرد را حذف کن.
- هر پاراگراف را حول یک وظیفه نگه دار.

## D. Write

- فارسی بومی و مستقیم؛
- جمله‌های نسبتاً کوتاه و متنوع؛
- اصطلاح فنی دقیق، بدون نمایش دانش؛
- لحن آرام، انسانی و غیرتبلیغاتی؛
- دلیل و tradeoff، نه فقط فهرست feature؛
- عدم اختراع claim، تجربه، quote یا نتیجه.

## E. Verify

1. factual diff
2. temporal diff
3. protected literal diff
4. voice review
5. surface fit
6. mechanics lint

## F. Report

برای تغییر فایل:

- چه چیزی ساختاری تغییر کرد؛
- چه چیزی عمداً دست‌نخورده ماند؛
- کدام fact یا context هنوز نیاز به تأیید دارد؛
- چه checkهایی اجرا شد.
