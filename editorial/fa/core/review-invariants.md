# Review invariants

خروجی نهایی باید این invariantها را حفظ کند:

## Semantic

- پیام مرکزی ثابت است.
- stance نویسنده تغییر نکرده است.
- شرط، استثنا و caveat حذف نشده است.
- رابطه‌ی علت و نتیجه معکوس نشده است.

## Factual

- عدد، تاریخ، نام و status تغییر نکرده است.
- feature یا نتیجه‌ی جدید اختراع نشده است.
- ownership تقویت یا تضعیف نشده است.
- uncertainty به قطعیت تبدیل نشده است.

## Temporal

- planned/current/historical با هم مخلوط نشده‌اند.
- عبارت relative بدون anchor وارد متن پایدار نشده است.

## Mechanical

- frontmatter، URL، code، placeholder و key سالم‌اند.
- Markdown و structured syntax خراب نشده است.

## Voice

- متن corporate، PR-like، بیش‌ازحد رسمی یا هیجانی نشده است.
- جمله‌ها ریتم انسانی دارند.
- تعریف از پروژه جای توضیح سازوکار را نگرفته است.

## Scope

- میزان تغییر با operation هم‌خوان است.
- context نامرتبط نشت نکرده است.
- recipe به قالب اجباری تبدیل نشده است.
