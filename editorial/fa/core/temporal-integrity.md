# Temporal integrity

متن باید بداند درباره‌ی چه زمانی حرف می‌زند.

## حالت‌ها

### timeless

برای اصول، معماری پایدار، راهنما و تعریف. از «الان»، «جدید»، «اخیراً» و «آخرین» بدون مرجع پرهیز کن.

### current

وضعیت جاری که باید با تاریخ یا نسخه‌ی منبع تأیید شود.

### historical

رویداد گذشته با تاریخ یا دوره‌ی مشخص.

### planned

کاری که تصمیم گرفته شده اما هنوز انجام نشده است.

### event-bound

متنی وابسته به release، کمپین، جلسه یا رویداد مشخص.

## قواعد

- current state را از حافظه یا متن قدیمی حدس نزن.
- «فعلاً» را در سندی که قرار است پایدار بماند حذف یا زمان‌دار کن.
- «نسخه‌ی جدید» را به version مشخص وصل کن.
- plan را با فعل آینده یا status روشن بنویس.
- گزارش تاریخی را با present tense مبهم بازنویسی نکن.
- تاریخ شمسی/میلادی و timezone را در متن حساس مبهم نگذار.

## نمونه

مبهم:

```text
در حال حاضر سیستم از موتور جدید استفاده می‌کند.
```

مرجع پایدار:

```text
سیستم محتوا را با موتور X پردازش می‌کند.
```

گزارش تاریخی:

```text
در بازطراحی نسخه‌ی ۳، موتور X جایگزین مسیر قبلی شد.
```

status report:

```text
وضعیت بررسی‌شده در ۱۱ ژوئیه‌ی ۲۰۲۶: مسیر پردازش بر پایه‌ی X است.
```
