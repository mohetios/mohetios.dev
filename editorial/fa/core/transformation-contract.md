# Transformation contract

## اصل

نوع عملیات تعیین می‌کند چه مقدار اجازه‌ی تغییر وجود دارد.

## عملیات

### proofread

فقط غلط، نشانه‌گذاری، فاصله و خطای واضح. ساختار و لحن را تغییر نده.

### light-edit

روانی و ایجاز محدود؛ ترتیب و صدای اصلی حفظ شود.

### structural-edit

ترتیب بخش‌ها، تکرار و جریان استدلال اصلاح شود؛ facts و stance ثابت بمانند.

### rewrite

بیان می‌تواند از نو ساخته شود، اما پیام، facts، مالکیت، uncertainty و محدودیت‌ها حفظ شوند.

### create

از brief و sourceها متن جدید بساز؛ چیزی خارج از evidence اختراع نکن.

### translate

معنا و جزئیات حفظ شوند؛ ساختار تا حد لازم برای فارسی طبیعی تغییر کند.

### localize

هدف، audience و effect حفظ شوند؛ wording، مثال و ریتم برای فارسی بازطراحی شود.

### adapt

یک artifact را برای surface دیگری بازسازی کن؛ copy-paste یا خلاصه‌ی مکانیکی نکن.

### summarize

اطلاعات کم‌اهمیت حذف شود، نه caveat و شرط حیاتی.

### expand

فقط با source یا reasoning مجاز گسترش بده؛ fact جدید نساز.

### audit

مشکل‌ها را مشخص کن و در صورت درخواست replacement بده؛ audit را خودکار به rewrite کامل تبدیل نکن.

## Freeze by default

- fact، عدد، تاریخ و نام
- ownership و attribution
- status و temporal mode
- URL، code، key، command و placeholder
- quote
- limitation و uncertainty
- intent و stance

## Scope drift

اگر task «غلط‌گیری» است، متن را marketing copy نکن. اگر task «کوتاه‌کردن» است، نظر نویسنده را تغییر نده. اگر task «دوستانه‌تر» است، اطلاعات جدید اضافه نکن.
