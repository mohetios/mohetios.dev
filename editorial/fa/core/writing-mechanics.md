# قواعد مکانیکی فارسی

## حروف

- `ی` و `ک` فارسی، نه `ي` و `ك` عربی.
- نیم‌فاصله در پیشوند و پسوندهای رایج: `می‌شود`، `نمی‌کند`، `آن‌ها`، `داده‌ها`.
- در identifier، URL، code و quote چیزی را normalize نکن.

## نشانه‌گذاری

- ویرگول فارسی: `،`
- نقطه‌ویرگول فارسی: `؛`
- پرسش فارسی: `؟`
- قبل از نشانه فاصله نگذار.
- بعد از نشانه یک فاصله بگذار، مگر پایان خط یا ساختار code.
- از تکرار `!!!`، `؟؟؟` و ellipsis نمایشی پرهیز کن.

## عدد

- در متن عمومی فارسی، عدد فارسی معمولاً خواناتر است.
- در code، version، path، command، IP، URL، schema و data literal عدد را تغییر نده.
- یک سند باید سیاست عدد ثابتی داشته باشد؛ ترکیب تصادفی نکن.

## تیتر

- تیتر وظیفه‌ی بخش را روشن کند.
- تیتر شاعرانه یا clickbait برای متن فنی نساز.
- nesting بیش از چهار سطح معمولاً نشانه‌ی ساختار ضعیف است.

## پاراگراف

- هر پاراگراف یک وظیفه‌ی اصلی داشته باشد.
- پاراگراف طولانی را فقط وقتی بشکن که جریان استدلال روشن‌تر شود.
- برای هر جمله bullet نساز؛ bullet برای مجموعه‌ی هم‌جنس است.

## Markdown و structured files

دست‌نخورده بمانند مگر task خلافش را بخواهد:

- frontmatter
- fenced code
- inline code
- HTML attributes
- component tags
- table alignment
- placeholders
- translation keys
- JSON keys
- YAML field names
