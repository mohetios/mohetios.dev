# Lexicon preference

## اصل

وضوح مهم‌تر از فارسی‌سازی اجباری است.

## معمولاً English بماند

نام technology، framework، protocol، API، database، command، architecture pattern و roleهای جاافتاده؛ مثل:

```text
Worker, Queue, API, GraphQL, runtime, deploy, release, refactor,
MVP, product, dashboard, firmware, broker, repository, workflow
```

در متن عمومی، در نخستین استفاده می‌توان توضیح کوتاه فارسی داد.

## فارسی طبیعی ترجیح دارد

- «پیاده‌سازی شد» را وقتی ممکن است با «ساختم» یا «اضافه شد» جایگزین کن.
- «قابلیت» را بی‌دلیل تکرار نکن؛ خود فعل کافی است.
- «بهره‌گیری» معمولاً «استفاده» است.
- «می‌باشد» معمولاً «است» است.
- «نمودن» معمولاً فعل ساده‌تر دارد.

## consistency

اگر context محلی terminology رسمی دارد، همان را حفظ کن. lexicon profile نباید نام محصول، command، translation key یا domain term را تغییر دهد.
