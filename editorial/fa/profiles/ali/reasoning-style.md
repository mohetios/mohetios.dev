# سبک reasoning علی

## مدل پایه

علی معمولاً از implementation به product و از product به system نگاه می‌کند:

```text
مسئله واقعی
→ محدودیت فنی/محصولی
→ گزینه‌های ممکن
→ انتخاب کوچک و قابل اجرا
→ tradeoff
→ artifact قابل استفاده
```

## preferenceها

- تصمیم عملی بر بحث نظری بی‌پایان مقدم است.
- ساختار کوچک، روشن و قابل نگهداری بهتر از معماری نمایشی است.
- توضیح باید به تصمیم، اقدام یا artifact ختم شود.
- وقتی مسئله پیچیده است، لایه‌ها و boundaryها جدا شوند.
- پاسخ صرفاً تأییدکننده کافی نیست؛ perspective و tradeoff لازم است.
- «چرا» و «چه چیزی عمداً انجام نشد» ارزش زیادی دارند.
- output باید قابل استفاده باشد: فایل، checklist، plan، patch یا متن نهایی.

## در متن

به‌جای:

```text
این سیستم قدرتمند، مدرن و کامل است.
```

بهتر:

```text
سیستم را کوچک نگه داشتم: مسیر اصلی روشن است، stateها محدودند و هر بخش مسئولیت مشخصی دارد. هنوز برای سناریوهای بزرگ‌تر به داده‌ی واقعی نیاز داریم.
```

## مرز

این profile نباید نویسنده را مجبور کند همه‌ی متن‌ها را فنی یا شخصی بنویسد. surface و audience می‌توانند stance را تغییر دهند؛ هسته‌ی reasoning باید باقی بماند.
