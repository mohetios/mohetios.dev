# rubric بازبینی

## hard gates

هر مورد زیر خروجی را رد می‌کند:

- factual claim جدید و بدون منبع
- تغییر ناخواسته‌ی frontmatter، link، code یا identifier
- roadmap به‌عنوان feature موجود
- overclaim امنیتی یا ناشناسی
- ownership اشتباه
- تغییر معنای فنی
- حذف limitation مهم
- quote یا citation ساختگی

## امتیاز ۱۰۰

### صحت و حفظ معنا — ۲۰

### نزدیکی به صدای هدف — ۲۰

### ساختار — ۱۵

### وضوح و اقتصاد متن — ۱۵

### کیفیت reasoning — ۱۰

### اصطلاحات فنی — ۱۰

### نگارش فارسی و Markdown — ۱۰

## راهنمای امتیازدهی

- امتیاز کامل: دقیق، طبیعی، روشن و قابل‌انتشار
- حدود ۷۵٪: یک یا دو ضعف کوچک
- حدود ۵۰٪: ضعف محسوس و نیاز به بازنویسی
- صفر: تغییر معنا یا شکست جدی

## تفسیر

| امتیاز | وضعیت |
|---:|---|
| ۹۵–۱۰۰ | آماده‌ی انتشار پس از factual check |
| ۸۵–۹۴ | قابل‌قبول؛ بازبینی انسانی کوتاه |
| ۷۰–۸۴ | نیازمند بازنویسی هدفمند |
| کمتر از ۷۰ | نامناسب |
| هر hard gate | رد |

## self-review

1. آیا این متن را می‌شد برای هر پروژه‌ی دیگری استفاده کرد؟
2. آیا مسئله و تصمیم concrete هستند؟
3. آیا محدودیت یا مرز نسخه روشن است؟
4. آیا جمله‌ای فقط برای خوب‌نشان‌دادن پروژه وجود دارد؟
5. آیا اصطلاحی بی‌دلیل ترجمه شده است؟
6. آیا «من/ما» مالکیت را درست نشان می‌دهد؟
7. آیا چیزی به facts اضافه شده است؟
8. آیا پایان اطلاعات تازه یا وضعیت واقعی می‌دهد؟
