# نمونه‌ی شروع معرفی Project One

Project One از یک سؤال ساده شروع شد: چطور می‌شود یک جریان مشخص را ساخت، بدون اینکه هویت کاربر یا جزئیات فرعی به مرکز تجربه تبدیل شود؟

نسخه‌ی اول قرار نیست همه‌ی مسئله را حل کند. تمرکز آن روی یک جریان محدود و قابل‌فهم است: دریافت ورودی، پردازش مسیر اصلی، پاسخ‌دادن و نگه‌داشتن حداقل state لازم برای ادامه‌ی کار. همین محدودکردن scope باعث شد بتوانم معماری و مدل نگه‌داری داده را دقیق‌تر طراحی کنم، به‌جای اینکه از ابتدا مجموعه‌ای از featureهای نیمه‌کاره کنار هم بگذارم.
