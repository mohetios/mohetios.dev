## یادداشت‌های مطالعه

_Freescale ARM Cortex-M Embedded Programming Using C Language_ زبان C را از تمرین به کار با دستگاه می‌برد. این جابه‌جایی مهم است: embedded مجبور می‌کند کد با register، timing، interrupt، نقشه حافظه و محدودیت‌های سخت‌افزار روبرو شود.

این کتاب کنار مطالب C می‌نشیند چون زبان را به ابزاری برای کنترل سیستم واقعی تبدیل می‌کند.

## ایده‌های مفید

### Registerها interface هستند

در embedded، interface اغلب یک register نگاشت‌شده به حافظه است. خواندن دقیق manual بخشی از برنامه‌نویسی می‌شود.

### Timing یک feature است

کد embedded درست فقط مقدار درست را تولید نمی‌کند. باید در زمان درست هم تولید کند.

### سیستم‌های کوچک به نظم بیشتر نیاز دارند

حافظه و observability محدود، ساختار را مهم‌تر می‌کند نه کم‌اهمیت‌تر.

## چرا نزدیک نگهش می‌دارم

پلی است از اصول C به مهندسی آگاه از سخت‌افزار. وقتی نرم‌افزار باید مستقیم با ماشین کنار بیاید مفید است.
