## یادداشت‌های مطالعه

_React Key Concepts_ کتاب تثبیت مفیدی است. شروع React آسان است، اما کار واقعی یاد گرفتن این است که کدام بخش‌های مدل باید ساده بمانند و کدام‌ها به ساختار عمدی نیاز دارند.

ارزش کتاب مقدمه دیگر JSX نیست. توجه مکرر به component، prop، state، effect، composition و حرکت داده در interface است.

## ایده‌های مفید

### Componentها مرز هستند

component خوب React فقط تکه بصری نیست. مرزی است دور رفتار، نیاز داده و تصمیم render.

### state به ownership نیاز دارد

خیلی از مشکل‌های React از ownership مبهم state می‌آید. سؤال معمولاً «state کجا می‌تواند زندگی کند؟» نیست؛ «کدام بخش interface مسئول تغییرش است؟»

### effectها باید مشکوک باشند

effect مفید است، اما جایی است که component ساده سخت‌تر فهمیده می‌شود. codebase React وقتی بهتر می‌شود که effect را نقطه integration بدانیم نه ابزار پیش‌فرض.

## چرا نزدیک نگهش می‌دارم

مرور عملی برای کار frontend. بحث React را قبل از framework بیشتر، state management بیشتر یا abstraction بیشتر به اصول برمی‌گرداند.
