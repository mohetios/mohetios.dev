## یادداشت‌های مطالعه

_Refactoring UI_ کتاب طراحی برای کسانی است که در جزئیات پیاده‌سازی فکر می‌کنند.

ایده قوی‌اش این است که طراحی interface بهتر می‌تواند از tacticهای تکرارپذیر بیاید. لازم نیست منتظر اعتماد هنری بمانی تا hierarchy، فاصله، تایپوگرافی، رنگ، عمق و empty state را بهتر کنی.

برای مهندس محصولی که اغلب نسخه اول ابزار را قبل از designer اختصاصی می‌سازد، به‌خصوص مفید است.

## ایده‌های مفید

### از feature شروع کن

کتاب علیه طراحی از layout خالی فشار می‌آورد. از کاری که interface باید انجام دهد شروع کن، بعد ساختاری را انتخاب کن که آن کار را پشتیبانی کند.

کار را به رفتار گره می‌زند نه تزئین.

### hierarchy قبل از styling

خیلی interfaceهای ضعیف به خاطر palette زشت نیستند. مبهم‌اند چون همه چیز وزن بصری یکسان دارد. اندازه، وزن، contrast و فاصله باید بگویند اول چه چیزی مهم است.

### سیستم بهتر از حدس زدن

مقیاس فاصله، مقیاس type، بازه رنگ و الگوی component انتخاب‌های تصادفی را کم می‌کند. سیستم کوچک صفحه بعدی را هم طراحی و هم پیاده‌سازی آسان‌تر می‌کند.

## چرا نزدیک نگهش می‌دارم

مرجع عملی برای تبدیل UI خام ساخته‌شده توسط developer به چیزی خوانا، عمدی و قابل ship. کمتر درباره سلیقه و بیشتر درباره دیدن سریع مشکل‌های رایج interface.
